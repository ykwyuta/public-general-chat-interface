# General Chat Interface

汎用 AI チャットインターフェース

---

## 要件定義書

### 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | General Chat Interface |
| 目的 | 汎用 AI チャット Web UI を提供する |
| 技術スタック | Next.js (App Router) / React / TypeScript / SQLite |
| 対象ユーザー | AI API を利用したチャットアプリを構築・利用したい開発者・エンドユーザー |

---

### 2. 機能要件

#### 2.1 チャット機能

| # | 機能 | 説明 |
|---|------|------|
| F-01 | メッセージ送信 | テキスト入力欄からメッセージを送信できる |
| F-02 | メッセージ受信 | AI からの返答をストリーミング形式でリアルタイム表示できる |
| F-03 | Markdown レンダリング | AI の返答中の Markdown（コードブロック・表・リストなど）を正しく描画する |
| F-04 | コードハイライト | コードブロックに構文ハイライトとコピーボタンを提供する |
| F-05 | メッセージ再生成 | AI の返答を再生成（リトライ）できる |
| F-06 | メッセージ編集 | ユーザーの送信済みメッセージを編集して再送信できる |
| F-07 | 会話リセット | 現在の会話をクリアして新規会話を開始できる |

#### 2.2 会話管理機能

| # | 機能 | 説明 |
|---|------|------|
| F-08 | 会話履歴一覧 | 過去の会話一覧をサイドバーに表示する |
| F-09 | 会話の新規作成 | 「新しいチャット」ボタンから新規会話を開始できる |
| F-10 | 会話の切り替え | サイドバーから過去の会話に切り替えられる |
| F-11 | 会話タイトル自動生成 | 最初のメッセージを元に会話タイトルを自動生成する |
| F-12 | 会話の削除 | 不要な会話を削除できる |
| F-13 | 会話の永続化 | 会話履歴を SQLite (better-sqlite3) に保存し、リロード後も復元できる |

#### 2.3 モデル・設定機能

| # | 機能 | 説明 |
|---|------|------|
| F-14 | モデル選択 | 使用する AI モデルを UI から切り替えられる |
| F-15 | システムプロンプト設定 | カスタムシステムプロンプトを設定できる |
| F-16 | プロバイダー設定 | 複数のAIプロバイダー（Anthropic, Gemini, Bedrock等）から使用するものを選択できる。APIキーはサーバーの環境変数で管理する |

#### 2.4 シナリオ（デモエージェント）機能

特定の設定（シナリオ）に基づき、あらかじめ定義された応答ルールや選択肢でやり取りできるデモエージェント（人工無能）機能を提供する。

| # | 機能 | 説明 |
|---|------|------|
| F-17 | シナリオ選択 | ユーザーが用意されたシナリオから会話相手を選択できる |
| F-18 | YAMLシナリオ定義 | 応答ルールや状態遷移をYAMLファイルで定義できる |
| F-19 | 選択肢ボタン表示 | シナリオの遷移条件に基づく選択肢をボタンとして表示し、ワンクリックで返信できる |
| F-20 | 条件分岐・状態管理 | ユーザーの選択に応じてシナリオ状態が遷移し、適切なテキストや選択肢を表示する |

#### 2.5 アーティファクト機能

AI の返答の中に含まれるコード・HTML・SVG 等を「アーティファクト」として抽出し、専用パネルに表示する。

| # | 機能 | 説明 |
|---|------|------|
| F-21 | アーティファクト自動検出 | AI 返答内のコードブロック（言語タグ付き）をアーティファクトとして自動抽出する |
| F-22 | アーティファクトパネル表示 | チャット領域の右側にアーティファクトパネルを表示する。アーティファクトが存在しない場合はパネルを非表示にし、チャット領域を全幅で使用する |
| F-23 | ファイルごとの展開／折りたたみ | 複数のアーティファクトがある場合、各ファイルを個別に展開・折りたたみできる |
| F-24 | アーティファクトのコピー | 各アーティファクトのコードをクリップボードにコピーできる |
| F-25 | HTML プレビュー | `html` 言語のアーティファクトは、コード表示とブラウザプレビュー（iframe）を切り替えられる |
| F-26 | ファイル名の推定・表示 | 言語タグやコード内容からファイル名（例: `index.html`, `component.tsx`）を推定して表示する |
| F-27 | アーティファクト一覧ナビゲーション | パネル上部に全アーティファクトの一覧（ファイル名リスト）を表示し、クリックでジャンプできる |
| F-28 | メッセージとアーティファクトのリンク | どのメッセージのターンで生成されたアーティファクトかを紐づけて管理する |

**対応アーティファクト種別**

| 種別 | 言語タグ例 | 表示方法 |
|------|-----------|---------|
| HTML | `html` | コード表示 ＋ iframe プレビュー |
| React / JSX | `jsx`, `tsx` | コード表示（ハイライト） |
| TypeScript / JavaScript | `ts`, `js`, `typescript`, `javascript` | コード表示（ハイライト） |
| CSS | `css`, `scss` | コード表示（ハイライト） |
| Python | `python`, `py` | コード表示（ハイライト） |
| SVG | `svg` | コード表示 ＋ SVG プレビュー |
| Markdown | `markdown`, `md` | Markdown レンダリング |
| 画像 | `image` | 画像表示 |
| その他 | 上記以外の言語タグ | コード表示（ハイライト） |

#### 2.7 タスク・タスクテンプレート機能

特定の目的と完了条件を持った「タスク」を作成し、複数の人間・LLMエージェントが参加してチャットを行う機能を提供する。

| # | 機能 | 説明 |
|---|------|------|
| F-36 | タスク作成・管理 | タイトル、目的、完了条件を指定してタスクを作成・管理できる |
| F-37 | マルチエージェント | 1つのタスクに複数のLLMエージェントと人間が参加し、発言できる |
| F-38 | メンション機能 | `@agent_name` の形式で特定のエージェントを指名して発言を促すことができる |
| F-39 | タスクテンプレート | よく使うタスク設定（参加エージェント構成含む）をテンプレートとして保存し、再利用できる |

#### 2.8 認証機能

| # | 機能 | 説明 |
|---|------|------|
| F-40 | ログイン | Google OAuth、および YAMLファイルで定義したデモ用アカウントによるログインが可能 |
| F-41 | ユーザー管理 | ログインユーザーごとのチャット履歴やタスク、テンプレートを分離して管理・保存できる |

#### 2.9 画像添付機能

メッセージ送信時に画像（JPEG/PNG/GIF/WebP）を添付し、LLMに送信できる。

| # | 機能 | 説明 |
|---|------|------|
| F-42 | 画像の添付 | ファイル選択により複数の画像をメッセージに添付できる |
| F-43 | 添付画像のプレビューと削除 | 送信前に添付した画像のサムネイルをプレビューし、削除できる |

#### 2.10 チャットテンプレート機能

特定の用途や目的に合わせたチャット環境をテンプレートとして構築し、再利用できる。

| # | 機能 | 説明 |
|---|------|------|
| F-44 | テンプレート作成・管理 | ウェルカムメッセージ、システムプロンプト、デフォルトMCP、コンテキストコンテンツを設定して保存できる |
| F-45 | ワークスペースへの自動展開 | テンプレートからチャットを開始する際、事前提供データをセッションごとのワークスペースに自動コピーする |

#### 2.11 MCP（Model Context Protocol）連携機能

外部のデータソースやツールと安全に連携し、LLMの機能を拡張する。

| # | 機能 | 説明 |
|---|------|------|
| F-46 | MCPサーバーの接続管理 | stdio（ローカル）やSSE（リモート）形式でMCPサーバーを登録・管理・接続できる |
| F-47 | ツールの動的拡張 | MCPサーバーが提供するツールをUIから選択し、LLMに公開して実行できる |
| F-48 | リソースのコンテキスト注入 | MCPサーバーが提供するデータをメッセージのコンテキストとして添付できる |
| F-49 | ツール実行の可視化 | チャット内でMCPツールが実行されたことを専用バッジとして視覚的に表示する |
| F-50 | 組み込みファイル入出力MCP | LLMが生成した内容の保存、ワークスペース内ファイルの読み込みを安全に行う |

#### 2.12 アーティファクト共有・通知（依頼）機能

特定のユーザーに対して、チャットで生成されたアーティファクトとメッセージを共有し、作業を依頼する。

| # | 機能 | 説明 |
|---|------|------|
| F-51 | メンション・アーティファクト指定 | `@username` と `#artifact_name` を用いて宛先と対象ファイルを指定してメッセージを送信できる |
| F-52 | アプリ内通知機能 | メンションされたユーザーにベルアイコンのバッジで通知し、一覧を表示する |
| F-53 | 依頼チャットの開始 | 通知から、共有されたアーティファクトとメッセージを引き継いだ新しいチャットを開始できる |
| F-54 | 依頼事項バナー表示 | 依頼チャットの画面上部に「依頼事項」「依頼者」「依頼日時」をバナー表示する |

### 3. 非機能要件

| # | 項目 | 要件 |
|---|------|------|
| NF-01 | パフォーマンス | 初回表示（FCP）3 秒以内 |
| NF-02 | ストリーミング | AI 応答をトークン単位でリアルタイム表示（待機感をなくす） |
| NF-03 | セキュリティ | API キーはサーバーの環境変数 (`.env.local`) で管理し、ブラウザには露出させない |
| NF-04 | アクセシビリティ | キーボード操作・スクリーンリーダー対応（WCAG 2.1 AA 準拠を目標） |
| NF-05 | 拡張性 | LLMProviderインターフェースを通じた設計により、AI プロバイダー（Anthropic・OpenAI 等）やローカルモック（シナリオエンジン等）を容易に差し替え可能な構造とする |

**主要な型定義（`types/index.ts`）**

```typescript
export type ArtifactKind = 'code' | 'html' | 'svg' | 'markdown' | 'image';

export interface Artifact {
  id: string;
  filename: string;
  language: string;
  kind: ArtifactKind;
  content: string;
  mimeType?: string; // 画像の場合のMIMEタイプ
  isExpanded: boolean;
}

export interface ImageAttachment {
  id: string;
  data: string;       // base64 encoded (without data URL prefix)
  mediaType: 'image/jpeg' | 'image/png' | 'image/gif' | 'image/webp';
  name: string;
}

export interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  images?: ImageAttachment[];
  artifacts: Artifact[];
  timestamp: Date;
}

export interface Conversation {
  id: string;
  title: string;
  messages: Message[];
  createdAt: Date;
  updatedAt: Date;
  scenarioId?: string;
  requestMessage?: string;
  requestSender?: string;
  requestCreatedAt?: Date;
}

export interface Settings {
  systemPrompt: string;
  model: string;
  /** 使用するLLMプロバイダーID */
  provider: string;
  theme: 'light' | 'dark';
}
```

### 9. 開発・ビルド

ローカル環境での開発・ビルド手順です。

#### 環境変数の設定

`.env.local.example` をコピーして `.env.local` を作成し、必要な情報を設定してください。
（NextAuthの秘密鍵 `AUTH_SECRET` や各LLMプロバイダの APIキーなどが含まれます）

```bash
cp .env.local.example .env.local
```

#### デモ用ユーザー設定

デモ用アカウントでログインするための `users.yaml` を作成します。

```bash
cp data/users.yaml.example data/users.yaml
```

#### モデルの設定

利用可能なLLMモデルは `public/models.yaml` で一元管理しています。**ビルドなし・サーバー再起動なし**で変更が反映されます。

```yaml
# public/models.yaml

models:
  # Anthropic Claude
  - id: claude-sonnet-4-6
    name: Claude Sonnet 4.6
    provider: anthropic

  # Google Gemini
  - id: gemini-3-flash-preview
    name: Gemini 3 Flash Preview
    provider: gemini

  # AWS Bedrock
  - id: amazon.nova-pro-v1:0
    name: Amazon Nova Pro
    provider: bedrock
```

| フィールド | 説明 |
|-----------|------|
| `id` | LLM APIに渡すモデルID（プロバイダーのドキュメントを参照） |
| `name` | UI上に表示される名前（任意の文字列） |
| `provider` | `anthropic` / `gemini` / `bedrock` のいずれか |

> **注意:** 環境変数でAPIキーが設定されていないプロバイダーのモデルは自動的に非表示になります。
>
> | provider | 必要な環境変数 |
> |----------|--------------|
> | `anthropic` | `ANTHROPIC_API_KEY` |
> | `gemini` | `GOOGLE_GENERATIVE_AI_API_KEY` |
> | `bedrock` | `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` |

モデルを追加・変更する場合は `public/models.yaml` を編集するだけです。コードの変更は不要です。
