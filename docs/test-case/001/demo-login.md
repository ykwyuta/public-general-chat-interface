# デモログイン機能 テストケース

## 1. 概要
本テストケースは、デモ認証機能（`data/users.yaml` に定義されたユーザー名・平文パスワードによる認証）が設計通りに動作するかを検証します。
PlaywrightによるE2Eテスト化を前提としています。

## 2. 前提条件
- `data/users.yaml` に以下のユーザーが定義されていること（`data/users.yaml.example` 参照）
  - ユーザー名: `admin`, パスワード: `password123`, 表示名: `管理者`
  - ユーザー名: `demo`, パスワード: `demo`, 表示名: `デモユーザー`

## 3. テストケース一覧

### TC-01: ログイン画面の初期表示確認
- **目的**: 未認証時にログイン画面が正しく表示されることを確認する。
- **手順**:
  1. 未認証の状態でアプリケーションのルートURL（`/`）にアクセスする。
- **期待結果**:
  - タイトル「General Chat Interface」が表示されること。
  - 「ユーザー名」の入力フィールドが表示されること。
  - 「パスワード」の入力フィールドが表示されること。
  - 「ログイン」ボタンが表示され、活性状態であること。

### TC-02: 正常系ログイン成功（Adminユーザー）
- **目的**: 存在するユーザー名と正しいパスワードでログインできることを確認する。
- **手順**:
  1. アプリケーションにアクセスし、ログイン画面を表示する。
  2. ユーザー名に `admin` を入力する。
  3. パスワードに `password123` を入力する。
  4. 「ログイン」ボタンをクリックする。
- **期待結果**:
  - ログイン処理が完了し、チャット画面（AppLayout）へ遷移すること。
  - ヘッダー右上にログインユーザーの表示名「管理者」が表示されること。

### TC-03: 正常系ログイン成功（Demoユーザー）
- **目的**: 別の存在するユーザー名と正しいパスワードでもログインできることを確認する。
- **手順**:
  1. アプリケーションにアクセスし、ログイン画面を表示する。
  2. ユーザー名に `demo` を入力する。
  3. パスワードに `demo` を入力する。
  4. 「ログイン」ボタンをクリックする。
- **期待結果**:
  - ログイン処理が完了し、チャット画面（AppLayout）へ遷移すること。
  - ヘッダー右上にログインユーザーの表示名「デモユーザー」が表示されること。

### TC-04: 異常系ログイン失敗（存在しないユーザー）
- **目的**: 存在しないユーザー名を入力した場合にエラーとなり、ログインできないことを確認する。
- **手順**:
  1. ログイン画面を表示する。
  2. ユーザー名に `invalid_user` を入力する。
  3. パスワードに `any_password` を入力する。
  4. 「ログイン」ボタンをクリックする。
- **期待結果**:
  - 画面遷移が発生しないこと。
  - 画面上に「ユーザー名またはパスワードが正しくありません」というエラーメッセージが表示されること。
  - パスワードの入力フィールドがクリアされること。

### TC-05: 異常系ログイン失敗（パスワード間違い）
- **目的**: 存在するユーザー名に対して誤ったパスワードを入力した場合にエラーとなり、ログインできないことを確認する。
- **手順**:
  1. ログイン画面を表示する。
  2. ユーザー名に `admin` を入力する。
  3. パスワードに `wrong_password` を入力する。
  4. 「ログイン」ボタンをクリックする。
- **期待結果**:
  - 画面遷移が発生しないこと。
  - 画面上に「ユーザー名またはパスワードが正しくありません」というエラーメッセージが表示されること。
  - パスワードの入力フィールドがクリアされること。

### TC-06: ログアウト処理
- **目的**: ログイン後に正しくログアウトでき、再度ログイン画面に戻ることを確認する。
- **手順**:
  1. `admin` ユーザーでログインし、チャット画面を表示する。
  2. ヘッダー右上にある「ログアウト」ボタンをクリックする。
- **期待結果**:
  - アプリケーション全体が未認証状態になり、ログイン画面へ遷移すること。
  - `sessionStorage` 内の認証情報（`general-chat-auth` 等）がクリアされていること。
